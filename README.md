# HIASM_Elements

Репозиторий содержит дополнительные элементы HiAsm и сопутствующие runtime-файлы. Каждый элемент хранится в отдельном каталоге и может иметь собственный README с описанием API, требований и ограничений.

## Структура элемента

Типовой каталог элемента:

```text
ElementName/
  README.md
  Element/<version>/
    *.ini
    *.pas
    *.ico
  win-x86/
  win-x64/
```

`Element/<version>` содержит файлы самого элемента. Каталоги `win-x86`, `win-x64`, `Template` и другие runtime-ресурсы являются необязательными и описываются в README конкретного элемента.

## Установка

Для установки элемента откройте каталог `Element/<version>` и перетащите его `.ini` на окно HiAsm. HiAsm добавит элемент в свою базу и автоматически разложит лежащие рядом файлы элемента по нужным каталогам, включая одноименные `.pas` и `.ico`.

Если пакет содержит несколько элементов, перетащите `.ini` каждого нужного элемента из одного каталога `Element/<version>`.

Если элемент поставляет runtime-файлы, DLL, SDK или шаблоны, их размещение описывается в README конкретного элемента. Обычно такие файлы не являются частью импорта `.ini` и должны лежать рядом с собираемым EXE или в архитектурных подкаталогах рядом с ним.

## Совместимость

Совместимость указывается в README каждого элемента. В этом репозитории могут встречаться элементы для разных веток компилятора: Delphi 4, старый FPC x32 и FPC2 x32/x64 в ANSI/Unicode-режимах.

Для элементов, которым заявлена работа со старым FPC через FPC2, имеется в виду среда HiAsm с установленным поверх комплектом FPC2: этот комплект переопределяет часть знаков условной компиляции. В чистом HiAsm на старом FPC такие элементы, скорее всего, работать не будут, если README конкретного элемента не содержит отдельной чистой проверки. Дублированный комплект FPC2: <https://disk.yandex.ru/d/9nHdCgy8OygKmA>. Профильная тема форума HiAsm: <https://forum.hiasm.com/topic/61538/0>.
